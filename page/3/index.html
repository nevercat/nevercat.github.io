java六大必须理解的问题（来自网络少许有误）

亘古不变学习些事
java

对于这个系列里的问题，每个学Java的人都应该搞懂。当然，如果只是学Java玩玩就无所谓了。如果你认为自己已经超越初学者了，却不很懂这些问题，请将你自己重归初学者行列。内容均来自于CSDN的经典老贴。

问题一：我声明了什么！

String s = “Hello world!”;

许多人都做过这样的事情，但是，我们到底声明了什么？回答通常是：一个String，内容是“Hello world!”。这样模糊的回答通常是概念不清的根源。如果要准确的回答，一半的人大概会回答错误。
这个语句声明的是一个指向对象的引用，名为“s”，可以指向类型为String的任何对象，目前指向”Hello world!”这个String类型的对象。这就是真正发生的事情。我们并没有声明一个String对象，我们只是声明了一个只能指向String对象的引用变量。所以，如果在刚才那句语句后面，如果再运行一句：

String string = s;

我们是声明了另外一个只能指向String对象的引用，名为string，并没有第二个对象产生，string还是指向原来那个对象，也就是，和s指向同一个对象。

问题二：”==”和equals方法究竟有什么区别？

==操作符专门用来比较变量的值是否相等。比较好理解的一点是：
int a=10;
int b=10;
则a==b将是true。
但不好理解的地方是：
String a=new String(“foo”);
String b=new String(“foo”);
则a==b将返回false。

根据前一帖说过，对象变量其实是一个引用，它们的值是指向对象所在的内存地址，而不是对象本身。a和b都使用了new操作符，意味着将在内存中产生两个内容为”foo”的字符串，既然是“两个”，它们自然位于不同的内存地址。a和b的值其实是两个不同的内存地址的值，所以使用”==”操作符，结果会是false。诚然，a和b所指的对象，它们的内容都是”foo”，应该是“相等”，但是==操作符并不涉及到对象内容的比较。
对象内容的比较，正是equals方法做的事。

看一下Object对象的equals方法是如何实现的：
boolean equals(Object o){

return this==o;

}
Object对象默认使用了==操作符。所以如果你自创的类没有覆盖equals方法，那你的类使用equals和使用==会得到同样的结果。同样也可以看出，Object的equals方法没有达到equals方法应该达到的目标：比较两个对象内容是否相等。因为答案应该由类的创建者决定，所以Object把这个任务留给了类的创建者。

看一下一个极端的类：
Class Monster{
private String content;
…
boolean equals(Object another){ return true;}

}
我覆盖了equals方法。这个实现会导致无论Monster实例内容如何，它们之间的比较永远返回true。

所以当你是用equals方法判断对象的内容是否相等，请不要想当然。因为可能你认为相等，而这个类的作者不这样认为，而类的equals方法的实现是由他掌握的。如果你需要使用equals方法，或者使用任何基于散列码的集合（HashSet,HashMap,HashTable），请察看一下java doc以确认这个类的equals逻辑是如何实现的。

问题三：String到底变了没有？

没有。因为String被设计成不可变(immutable)类，所以它的所有对象都是不可变对象。请看下列代码：

String s = “Hello”;
s = s + “ world!”;

s所指向的对象是否改变了呢？从本系列第一篇的结论很容易导出这个结论。我们来看看发生了什么事情。在这段代码中，s原先指向一个String对象，内容是”Hello”，然后我们对s进行了+操作，那么s所指向的那个对象是否发生了改变呢？答案是没有。这时，s不指向原来那个对象了，而指向了另一个String对象，内容为”Hello world!”，原来那个对象还存在于内存之中，只是s这个引用变量不再指向它了。
通过上面的说明，我们很容易导出另一个结论，如果经常对字符串进行各种各样的修改，或者说，不可预见的修改，那么使用String来代表字符串的话会引起很大的内存开销。因为String对象建立之后不能再改变，所以对于每一个不同的字符串，都需要一个String对象来表示。这时，应该考虑使用StringBuffer类，它允许修改，而不是每个不同的字符串都要生成一个新的对象。并且，这两种类的对象转换十分容易。
同时，我们还可以知道，如果要使用内容相同的字符串，不必每次都new一个String。例如我们要在构造器中对一个名叫s的String引用变量进行初始化，把它设置为初始值，应当这样做：
public class Demo {
private String s;
…
public Demo {
s = “Initial Value”;
}
…
}
而非
s = new String(“Initial Value”);
后者每次都会调用构造器，生成新对象，性能低下且内存开销大，并且没有意义，因为String对象不可改变，所以对于内容相同的字符串，只要一个String对象来表示就可以了。也就说，多次调用上面的构造器创建多个对象，他们的String类型属性s都指向同一个对象。
上面的结论还基于这样一个事实：对于字符串常量，如果内容相同，Java认为它们代表同一个String对象。而用关键字new调用构造器，总是会创建一个新的对象，无论内容是否相同。
至于为什么要把String类设计成不可变类，是它的用途决定的。其实不只String，很多Java标准类库中的类都是不可变的。在开发一个系统的时候，我们有时候也需要设计不可变类，来传递一组相关的值，这也是面向对象思想的体现。不可变类有一些优点，比如因为它的对象是只读的，所以多线程并发访问也不会有任何问题。当然也有一些缺点，比如每个不同的状态都要一个对象来代表，可能会造成性能上的问题。所以Java标准类库还提供了一个可变版本，即StringBuffer。

问题四：final关键字到底修饰了什么？

final使得被修饰的变量”不变”，但是由于对象型变量的本质是“引用”，使得“不变”也有了两种含义：引用本身的不变，和引用指向的对象不变。

引用本身的不变：
final StringBuffer a=new StringBuffer(“immutable”);
final StringBuffer b=new StringBuffer(“not immutable”);
a=b;//编译期错误

引用指向的对象不变：
final StringBuffer a=new StringBuffer(“immutable”);
a.append(“ broken!”); //编译通过

可见，final只对引用的“值”(也即它所指向的那个对象的内存地址)有效，它迫使引用只能指向初始指向的那个对象，改变它的指向会导致编译期错误。至于它所指向的对象的变化，final是不负责的。这很类似==操作符：==操作符只负责引用的“值”相等，至于这个地址所指向的对象内容是否相等，==操作符是不管的。

理解final问题有很重要的含义。许多程序漏洞都基于此——final只能保证引用永远指向固定对象，不能保证那个对象的状态不变。在多线程的操作中,一个对象会被多个线程共享或修改，一个线程对对象无意识的修改可能会导致另一个使用此对象的线程崩溃。一个错误的解决方法就是在此对象新建的时候把它声明为final，意图使得它“永远不变”。其实那是徒劳的。

问题五：到底要怎么样初始化！

本问题讨论变量的初始化，所以先来看一下Java中有哪些种类的变量。
1. 类的属性，或者叫值域
2. 方法里的局部变量
3. 方法的参数

对于第一种变量，Java虚拟机会自动进行初始化。如果给出了初始值，则初始化为该初始值。如果没有给出，则把它初始化为该类型变量的默认初始值。

int类型变量默认初始值为0
float类型变量默认初始值为0.0f
double类型变量默认初始值为0.0
boolean类型变量默认初始值为false
char类型变量默认初始值为0(ASCII码)
long类型变量默认初始值为0
所有对象引用类型变量默认初始值为null，即不指向任何对象。注意数组本身也是对象，所以没有初始化的数组引用在自动初始化后其值也是null。

对于两种不同的类属性，static属性与instance属性，初始化的时机是不同的。instance属性在创建实例的时候初始化，static属性在类加载，也就是第一次用到这个类的时候初始化，对于后来的实例的创建，不再次进行初始化。这个问题会在以后的系列中进行详细讨论。

对于第二种变量，必须明确地进行初始化。如果再没有初始化之前就试图使用它，编译器会抗议。如果初始化的语句在try块中或if块中，也必须要让它在第一次使用前一定能够得到赋值。也就是说，把初始化语句放在只有if块的条件判断语句中编译器也会抗议，因为执行的时候可能不符合if后面的判断条件，如此一来初始化语句就不会被执行了，这就违反了局部变量使用前必须初始化的规定。但如果在else块中也有初始化语句，就可以通过编译，因为无论如何，总有至少一条初始化语句会被执行，不会发生使用前未被初始化的事情。对于try-catch也是一样，如果只有在try块里才有初始化语句，编译部通过。如果在catch或finally里也有，则可以通过编译。总之，要保证局部变量在使用之前一定被初始化了。所以，一个好的做法是在声明他们的时候就初始化他们，如果不知道要出事化成什么值好，就用上面的默认值吧！

其实第三种变量和第二种本质上是一样的，都是方法中的局部变量。只不过作为参数，肯定是被初始化过的，传入的值就是初始值，所以不需要初始化。

问题六：instanceof是什么东东？

instanceof是Java的一个二元操作符，和==，>，<是同一类东东。由于它是由字母组成的，所以也是Java的保留关键字。它的作用是测试它左边的对象是否是它右边的类的实例，返回boolean类型的数据。举个例子：

String s = “I AM an Object!”;
boolean isObject = s instanceof Object;

我们声明了一个String对象引用，指向一个String对象，然后用instancof来测试它所指向的对象是否是Object类的一个实例，显然，这是真的，所以返回true，也就是isObject的值为True。
instanceof有一些用处。比如我们写了一个处理账单的系统，其中有这样三个类：

public class Bill {//省略细节}
public class PhoneBill extends Bill {//省略细节}
public class GasBill extends Bill {//省略细节}

在处理程序里有一个方法，接受一个Bill类型的对象，计算金额。假设两种账单计算方法不同，而传入的Bill对象可能是两种中的任何一种，所以要用instanceof来判断：

public double calculate(Bill bill) {
if (bill instanceof PhoneBill) {
//计算电话账单
}
if (bill instanceof GasBill) {
//计算燃气账单
}
…
}
这样就可以用一个方法处理两种子类。

然而，这种做法通常被认为是没有好好利用面向对象中的多态性。其实上面的功能要求用方法重载完全可以实现，这是面向对象变成应有的做法，避免回到结构化编程模式。只要提供两个名字和返回值都相同，接受参数类型不同的方法就可以了：

public double calculate(PhoneBill bill) {
//计算电话账单
}

public double calculate(GasBill bill) {
//计算燃气账单
}

所以，使用instanceof在绝大多数情况下并不是推荐的做法，应当好好利用多态。

java方向及学习方法
java分成J2ME（移动应用开发），J2SE（桌面应用开发），J2EE(Web企业级应用)，所以java并不是单机版的，只是面向对象语言。建议如果学习java体系的话可以这样去学习：
第一阶段：Java基础，包括java语法，面向对象特征，常见API，集合框架； 第二阶段：java界面编程，包括AWT，事件机制，SWING，这个部分也可以跳过，用的时候再看都能来及；
第三阶段：java API：输入输出，多线程，网络编程，反射注解等，java的精华部分； 第四阶段：数据库SQL基础，包括增删改查操作以及多表查询；
第五阶段：JDBC编程：包括JDBC原理，JDBC连接库，JDBC API，虽然现在Hibernate比JDBC要方便许多，但是JDBC技术仍然在使用，JDBC思想尤为重要； 第六阶段：JDBC深入理解高级特性：包括数据库连接池，存储过程，触发器，CRM思想；
第七阶段：HTML语言学习，包括HTML标签，表单标签以及CSS，这是Web应用开发的基础； 第八阶段：JavaScript脚本语言，包括javaScript语法和对象，就这两个方面的内容；
第九阶段：DOM编程，包括DOM原理，常用的DOM元素以及比较重要的DOM编程思想； 第十阶段：Servlet开发，从此开始踏入java开发的重要一步，包括XML，Tomcat服务器的安装使用操作，HTTP协议简单理解，Servlet API等，这个是java web开发的基础。
第十一阶段：JSP开发：JSP语法和标签，自定义标签，EL,JSTL库了解以及MVC三层架构的设计模式理念； 第十二阶段：AJAX开发：AJAX原理，请求响应处理，AJAX开发库；
第十三阶段：轻量级框架，三大框架之一Struts框架的学习，自此踏入java web开发的精华部分，包括Struts体系架构，各种组件，标签库和扩展性的学习； 第十四阶段：Hibernate框架学习，三大框架之一，包括检索映射技术，多表查询技术，缓存技术以及性能方面的优化；
第十五阶段：Spring框架的学习，三大框架之一，包括了IOC,AOP,DataSource，事务，SSH集成以及JPA集成； 最后呢，还有些java的技术，包括EJB3.0等，可以选择学习，与三大轻量级框架相比，EJB就是当之无愧的重量级了。

MyEclipse的使用注意事项和程序的断点调试以及常用快捷键

亘古不变学习些事
java 
   放寒假之前网络中心的许可老师给我布置了个任务，寒假回去了解下“百度知道”的业务模式，开学后给我分配一台服务器用jsp的hibernate框架部署一个开源的问答系统（问答系统？想起了华科冰岩的iknow，不过iknow貌似是用php写的）。可惜寒假玩得太疯了基本把这事给抛在脑后了，现在数着日子差不多只有一个星期就开学了吧，开始准备收心了！好长时间手生没敲java代码，于是首先复习了一下，总结了一下自己平时使用Myeclipse进行开发时积累起来的一些技巧，主要是一些注意事项、个人用得最多的快捷键和程序的调试什么的，不过大神们请忽略这篇文章。。- -！

   1、创建Workspace的名字的时候千万不要使用中文或者带空格。否则你之后运行程序时错都不知道自己错在哪儿了，虽然之后也可以处理，但比较麻烦，要养成一个好的习惯。

   2、创建Project的时候，指定的java运行环境版本一定要和编译器的版本相同或者更高，否则就会报错。（修改jre的方法：一是可以在创建Project的时候直接配置是使用默认的还是自己指定的jdk[一般是指定jdk，包含了开发环境和运行环境jre]；二是可以在工程名上右键——Properties[快捷键Alt+Enter]——Java Build Path——Libraries。修改java编译器的方法：工程名上右键——Properties[快捷键Alt+Enter]——Java Compiler）

   3、创建class的时候一定要有包名，无论是写简单的程序还是复杂的工程，习惯很重要！

   4、必须先打断点才能进入debug模式。

   5、debug时的快捷键：

F5：跳入 step into
F6：跳过 step over
F7：跳出 step return
drop to frame：跳到当前方法的第一行（一般是在不小心按F6按快了超过了你想看的那一行时用）
resume：跳到下一个断点（如果没有下一个，则运行完整个程序。如果当前方法有几百几千几万行，一行一行的F6太慢想直接跳到下一个方法的话可在下一个方法开头再打一个断点，然后resume）
watch：观察变量或表达式的值（选中要看的变量或者表达式然后右键）

   6、断点注意事项：

(1)断点调试完成后，一定要在breakpoints视图中清除所有断点！
(2)断点调试完成后，一定要记得结束运行断点的JVM，否则你每调试一次开一个JVM最后机子卡死了没保存代码都不知道是怎么死的！

   7、常用快捷键：

内容提示：Alt+/ （老版本的MyEclipse默认内容提示的快捷键是Alt+Space，因为常常和输入法的切换快捷键冲突需要手动修改，新版的MyEclipse已默认为Alt+/。修改快捷键方法：Window——Preferences，然后输入keys即可进入设置快捷键界面）
快速修复：Ctrl+1 （当出现错误时，想都不用想，首先先按一下Ctrl+1再说，让其自动修复一下，若修复不了再自己找原因修复）
导包：Ctrl+Shift+O （当知道错误原因是没导入相应的包时可直接按这个，免得Ctrl+1快速修复需要按多次）
格式化代码块：Ctrl+Shift+F （必须要记住的快捷键！）
向前向后：Alt+左右方向键 （在调试或者查看源代码的时候，到处跳想返回或者前进跟踪源代码的时候很有效！）
添加注释：Ctrl+Shift+/
除去注释：Ctrl+Shift+\ （这2个就不用说了吧，记住要先选中需要注释的代码）
查看方法说明：F2 （鼠标移到方法名上出现说明然后Press “F2” for focus查看全部的说明）
重置透视图 （当窗口版块拖乱了时候想要恢复默认时，Window——Reset Perspective，确定即可。记住不要点到了Save Perspective As…）
更改为大写：Ctrl+Shift+X （例如常量需要大写，选中按快捷键即可）
更改为小写：Ctrl+Shift+Y
复制行：Ctrl+Alt+向下键 （注意有些不能用。不需要选中。默认复制光标所在行）
交换两行代码顺序：Alt+上下方向键
查看类的继承关系：Ctrl+T （鼠标放在类上按快捷键就会出现该类的父类和子类继承树）
查看源代码：(1)Ctrl+点击 （按住ctrl，鼠标移到对象上变成收到形状然后点击。有时要手动导入源代码，例如jdk里面的src.zip）
(2)Ctrl+Shift+T （当没写类的时候，然后输入想看类的源代码名称搜索）
查看所有快捷键：Ctrl+Shift+L （列出MyEclipse中所有的快捷键，但个人觉得掌握以上提要的常用的即可，太多了也记不住）

唧唧歪歪生活些事 
    寒假回来六天了，前三天玩穿了仙剑五前传，最后也算是被瑾轩和瑕的大悲剧结局感动得稀里哗啦；后三天无所事事，看了几部电影听了几首歌就过去了。突然发现这应该是我大学时光的倒数第二个寒假了。

    “诶，都长这么大了啊？”“读大几了啊？”“以后准备做什么啊？”“有女朋友了没啊？” “给你介绍几个吧？” “想什么时候结婚啊？” “那个谁谁跟你年纪差不多都有娃了诶？” 呵呵，果不其然，还没到正式过年亲戚朋友就已经这样了。不过说实在的，今天下半年也就是大四上去实习完恐怕工作随之也就会确定下来，到时候估计又会做些艰难的人生抉择，像几年前高考失利选择复读一样。一直以为我还很小，工作是件离我很遥远很遥远的事，读书读了这么多年，虽不喜欢学习，竟也不知道我除了学习外还能做些什么。

    大学读计算机专业本来就不是我的理想，阴差阳错的来到的一个文科性质的学校读一个工科专业，尤其不喜欢数学的我更是被计算机专业课程折磨得半死，更是不喜欢编写眼花缭乱的代码。说到专业课程，前不久考试周最后一门考的操作系统也是我深恶痛绝的，哈哈，这又不得不说在复习的过程中，由于复习得太忧伤导致一怒之下去把我们院的新版网站黑了，其实黑的过程很没技术含量，很偶然的找到的一个漏洞然后就侵入后台把首页替换掉了。再小小的得瑟下。


    大二的时候发现自己对网络方向还是有点兴趣，就在那方面投入了点精力，前前后后接触了几种编程语言，.net，php，jsp，每个都了解了点，但都没深入，我就是个不能持之以恒的人。asp.net是跟着陈利老师的一个项目用到的，但当时正好处于厌恶编程的迷茫期，本来就不喜欢复杂的C++，当然对C#也是很排斥。对php感兴趣不仅是因为它开源更是因为去年买了自己的第一个域名，用wordpress搭建了这个个人博客，起初还兴致盎然的折腾来折腾去，还专门去图书馆借了一些关于php的书来看，刚准备正式确定php作为我以后发展的web后台语言时，半路又杀出了个jsp。因为IRIS工作室和数据库课程设计的需要，当时暂时放下了php，又转去弄jsp，最后用jsp+mysql做了个客房预订系统，因为本来就有一点java基础，转java web开发确实还比较容易，所以对jsp并没asp.net那么排斥。当时做这个系统时，组内其他成员都对前端没什么经验，我就同时负责了系统的前端设计，这个时候我又发现我对前端设计也有一定的兴趣，完成这个项目时翻了很多javascript和jquery的书，还参考了一些html5+css3的技术，发现原来前端比后台更好玩。记得有次跟一同学聊天，他说我好厉害啊，什么都懂什么都会，我汗颜 - -！我只是什么都只懂点鸡毛蒜皮，都只停留在表层，后来他说我以后适合去当PM，我当时呵呵一笑了。

    所以我也不知道我应该往什么方向发展了。去年暑假在盛天网络实习了一段时间，反正就感觉程序猿确实很苦逼，可是计算机专业的不去当程序猿又能做什么咧，技术不愿意做，产品又做不来。或者我干脆老老实实的去当一个老师吧。

    有好些时候事情不会朝着你所想的那样发展，我们需要做的只是顺其自然。下学期的事情也比较多，一个软考一个专四，都是money要打水漂的节奏，然后暑假的时候看能不能去找个前端工程师的实习岗位吧，这学期重点就放在前端设计上。

    前些天玩游戏貌似把鼠标玩坏了，按键总有点不灵，单击老变成双击，看来罗技的鼠标也不靠谱。然后在逛京东的时候，突然有点想买单反的冲动，翻翻网银，发现就剩下上学期得的的奖学金还没动了，然后看今年还可以拿到多少压岁钱，都这么大了还得压岁钱的表示很汗颜啊，O__O"…要不然过完年再一冲动就可以入手个入门级的单反了！

写在最开始的

唧唧歪歪生活些事 
    咳咳。不知道大家现在是不是还在学校，十一有些什么打算呢，回家还是出游还是宅学校，反正我现在已经躺在家里舒服的大床上了，一边吃着东西一边敲这些字在。先小小的得瑟下。

    得瑟完了废话不多说就直接切入正题吧。

    首先很感谢大家选择我加入你们，感谢大家的信任，然后我肯定是不会辜负大家的期望的，相信我们组在一起会是最好的团队。我希望我们团队可以走到最后成为最后笑的那一个，当然自信点，我们必须是，我们就是！

    说点心里话，其实从一开始我就有这么个想法，在大学期间要做一个属于自己的产品，它可以是个网站，可以是个app，也可以是个其他的什么东西，不管做得好不好，但只要我去做了，它就是成功的，或许做的东西在别人看来一文不值，很逊很垃圾，很傻很天真，但对于自己却意义重大，尤其是做这个的过程和感觉是不可比拟的。所以之前我也在做这方面的尝试，包括从大一加入锐意实验室（一个很古老的故事），最终实验室因为种种原因解散了；后来大二的时候跟着陈利老师（你们应该都知道吧）酱油的负责了一个科研项目，当然那个项目最后做得很失败，最终草草的收尾了，只做了个雏形出来就夭折了，当时还很对不起陈老师的；再后来加入IRIS团队（也许你们11级有部分人知道这个团队吧，它也是主要由我们计院的同学组建起来的一个web开发团队，当然我们主要是用jsp来开发，java你们貌似还没开始学），这个还在进行中。总而言之就是我的大学已经过半了，却还是很失败，所以从进大学开始的那个最初梦想还在，然后，现在，盛天来了。

    然而就在不久前，就在这个浮躁的暑假，我因为一个偶然的机会有幸在盛天实习了一个月，总体对这个公司的感觉还是蛮好的，当时是在研发部做些酱油的工作，很简单很无聊，但学到了不少东西，包括简单的了解了一个互联网公司的运营方式，以及大致明白了一个项目从无到有产生的一个过程。这次盛天来了，我觉得这的确是一次少有的机遇，我在大学的时光也不多了，就大三这一年，大四前半年会出去实习，后半年会忙着毕业论文毕业设计忙着毕业伤感忙着说再见，能做事的时间数数看也就大三了，所以我还是很想把握住这个机会的。如果没有梦想，那和咸鱼有什么分别！这句话我一直觉得很励志，之前在小米的招聘会上小米的口号也是这个，咸鱼也有梦想，也梦想着有朝一日可以翻身，我也需要有一个机会来翻身，来证明自己的大学并没有白过……

    反过来说，这个对于你们的意义则更加重大，相信你们和我也对未来抱着一些美好的憧憬吧，而你们比我年轻，比我有着更多的机会，有着更好的资源，所以我希望从今天开始，从此刻开始，我们这个团队可以向着我们的梦想去奋斗！

    额，貌似扯远了。回归正题吧。

    昨天晚上开完会，由于吴睿和龙雪玲没去，估计也不太清楚到现在到底是个什么情况，我先大致的说说吧。

    我们要做的是，从最开始的一个idea到最后一个完整的产品上线，我们要以一个产品的形式来做这个，而不是简单的去做一个网站。当时陈浩然老师（我们以后的指导老师，盛天网络的产品经理）给我们提供了3个选题，这3个在我看来本质上是一样的，就是做个垂直类网站（什么是垂直类网站，就是单一细分的一种网站，针对某一特定人群做一个有一个特定主题的网站，具体请百度谷歌之。垂直类网站是很有发展前景的在目前看来），陈老师的3个选题是：1、垂直类导购网站（类似美丽说）；2、垂直类应用商店（类似机锋网）；3、垂直类自助结伴旅游社区（类似hicafe）。我和王瑞当时的选择是第三个，垂直类自主结伴旅游社区，我们的想法是做垂直类的SNS社交平台会比做单纯的做一个垂直类卖东西的更加有吸引力，创意会更多，局限小点。当然大家如果还有什么异议可以随时提，这个是可以更换选题的，甚至可以有自己的idea，自己想去做个什么类型的网站，都可以提出来，也鼓励这样哈。一旦选题确定下来我们就要开始着手做了。

    估计大家到现在也是迷迷糊糊的，不知道要做些什么前期工作，我就把我们现在需要做的先梳理下吧。

1、 我们需要有一个霸气的团队名。要么文艺，要么213，就是不能普通！像之前我加入的团队锐意啊，IRIS啊，这样的，可以是中文名也可以是英文名，也可以两者都有。

2、 确定下我们的选题后我们就要开始想我们要做的网站的网站名，同样需要霸气十足。这个名字当然只是前期的一个拟定名，在后期产品快成型的时候我们也可以根据具体实际更改，但项目全程我们都需要有个名字。也是中英文都可以有，甚至包括域名的选择。

3、 还是确定后选题后的事，我们确定后需要收集大量资料，国内外有关这个选题的已有网站资料以及它们的风格以及它们的创意，我们先要总结别人的成功之处才可能有自己的创新之处。

4、 想自己的创意。凡产品必有自己的创新，我们不希望只是复制别人的网站，别人做成什么样的我们就做成什么样，我们要以一个高的标准来要求自己，这里的创意可以很小，可以是在别人的基础上的微小创新，但这个必须要有，至少要有一个创意闪光点，告诉别人，我们和你们不一样。

5、 初期需要掌握4种软件，①axure；②photoshop；③excel；④ppt。当然这些只是初期的基本的，等以后后台语言确定下来我们会确定其他的开发软件以及数据库软件。我们可以分工来掌握这些，因为以后我们的工作应该会是分工的，有人做整体的构架，有人做前端页面，有人做后台代码，有人做配图，有人做设计等等等等，但我希望既然都是学习，都是从无到有，我们都掌握一些会更好，因为你做设计你不可能一直都做设计，你说不定以后做其他项目的时候又转去做后台开发了呢。

6、 大约在国庆节过后，也就是7号，我们要有一个初步的定稿，包括做什么，怎么做，确定目标用户，使用场景，甚至画出线稿图，竞争对手的分析，以及我们的创新点。然后14号之前修订完最后的定稿，形成一个完整的商业计划书。

    我是不是很啰嗦，好吧，那就不扯了，大家有什么开始可以在群里讨论，如果还有什么不清楚的，尽量问，不要还没搞清楚要做什么就胡乱的去行动结果最后发现南辕北辙。我希望我们每个人都可以有自己的想法，那么，让我们一块为梦想奋斗吧！梦想从这儿开始~！
